# my-shop-bot MVP 需求文档 (PRD)

## 一、产品一句话定位 (Elevator Pitch)

> **"一个关键词进去，一篇能打的英文 SEO 文章出来。专为出海卖家造的 AI 内容军火库。"**

---

## 二、核心用户画像

| 维度 | 描述 |
|------|------|
| **身份** | 中国出海独立站卖家（Shopify/WordPress），1-10 人小团队 |
| **赛道** | 家居用品 (Home Decor & Patio)、户外露营 (Camping & Outdoor) |
| **核心痛苦** | 英文不够好，写不出 native 级别的 SEO 文章；不懂 H2/H3 结构化标签；不了解 GEO（生成式引擎优化）新规则；每篇文章耗时 3-5 小时 |
| **预期价值** | 10 分钟内获得一篇结构完整、可直接发布的英文 SEO 博客 or Listing |
| **典型场景** | 上新一款 pop-up beach tent，需要一篇 2000 词的博客 + 一段高转化 Listing 描述 |

## 三、MVP 核心功能清单 (P0 — 第一版只做这些)

### 3.1 用户流程（极简四步）

```
[输入关键词] → [选择生成类型] → [AI 生成内容] → [一键复制]
```

### 3.2 功能拆解

#### P0-1：关键词输入

- 单个输入框，placeholder 示例：`pop-up beach tent`
- 支持输入 1-3 个关键词（逗号分隔）
- 无需登录，打开即用

#### P0-2：生成类型选择

两个大按钮，二选一：

| 类型 | 说明 | 输出规格 |
|------|------|---------|
| 📝 **SEO Blog Article** | 长文博客 | 1500-2500 词，含 H1/H2/H3 结构，Meta Title，Meta Description，自然嵌入关键词 |
| 🏷️ **Product Listing** | 产品详情页文案 | 含标题、5 条 Bullet Points、产品描述（300-500 词，含 H2 子标题）、SEO Meta |

#### P0-3：AI 内容生成引擎

**输出规范（硬编码进 Prompt，用户不可改）：**

- **语言**：美式英语，native speaker 语感，不准出现中式英语
- **SEO 结构**：
  - Blog：1 个 H1 → 4-6 个 H2 → 每个 H2 下 1-3 个 H3
  - Listing：标题 ≤ 200 字符 → Bullet Points × 5 → Description 含 2-3 个 H2
- **GEO 合规**：
  - 段落开头直接回答用户可能的搜索意图（适配 AI Overview 抓取）
  - 关键数据用结构化列表/表格呈现（适配 Featured Snippet）
  - 自然嵌入 "According to..." / "Based on..." 等权威引用句式
  - 每个 H2 段落首句即为该段核心观点（适配 SGE 摘要提取）
- **关键词密度**：主关键词出现 4-8 次，自然分布，不堆砌
- **CTA**：文末自动加一段通用转化引导段落
- **输出格式**：纯 Markdown，H 标签清晰可辨

**垂直赛道增强（内置知识）：**

- 家居用品：自动引用常见材质（rattan, teak, poly lumber）、场景词（patio, backyard, living room）、买家关注点（weather-resistant, UV-protected, easy assembly）
- 户外露营：自动引用装备参数（denier, R-value, lumens）、场景词（backcountry, car camping, overlanding）、买家关注点（ultralight, packable, waterproof rating）

#### P0-4：输出与复制

- 生成结果实时流式输出（打字机效果）
- 左侧：Markdown 源码（可编辑）
- 右侧：渲染预览（所见即所得）
- 顶部一个大按钮：**「📋 Copy Markdown」** — 一键复制全文到剪贴板
- 底部一个按钮：**「🔄 Regenerate」** — 不满意就重来

### 3.3 技术约束

| 项目 | 决策 |
|------|------|
| **前端** | 纯静态 SPA，部署在 likecamping.com/tools/seo-writer 或独立子域名 |
| **后端** | 仅一个 API 端点：`POST /api/generate` → 调用 LLM → 流式返回 |
| **LLM** | 阶跃星辰 API 或 OpenAI API，单一模型，不做模型选择 |
| **登录系统** | ❌ 第一版不做 |
| **数据库** | ❌ 第一版不做，不存任何用户数据 |
| **付费** | ❌ 第一版不做，完全免费/内部使用 |
| **速率限制** | 简单的 IP 级别限频（防滥用），10 次/小时 |

## 四、被砍掉的进阶功能 (Phase 2 — 第一版坚决不做)

| 功能 | 理由 | Phase 2 优先级 |
|------|------|---------------|
| 🔐 用户注册/登录 | MVP 阶段零摩擦优先，加登录就流失一半人 | P1 |
| 📜 生成历史记录 | 需要数据库，复杂度翻倍 | P1 |
| 💳 付费订阅/次数包 | 先验证产品价值再谈收费 | P1 |
| 📦 批量关键词导入 (CSV) | 高级功能，等单篇验证通后再做 | P1 |
| 🔌 一键发布到 Shopify | 需要 OAuth 对接，工程量大 | P2 |
| 🔌 一键发布到 WordPress | 同上 | P2 |
| 🌐 多语言生成（德/法/西/日） | 先把英文做到极致 | P2 |
| 📊 SEO 评分/关键词密度分析 | 锦上添花，不是核心 | P2 |
| 🖼️ AI 配图生成 | 依赖图像模型，成本和复杂度高 | P3 |
| 🧠 竞品文章分析/改写 | 需要爬虫+对比引擎 | P3 |
| 📈 SERP 排名追踪 | 完全不同的产品方向 | P3 |
| 🤖 API 开放给第三方 | 等商业模式验证后 | P3 |
| 🎨 自定义品牌语气/模板 | Phase 1 用硬编码 Prompt 足够 | P2 |
| 📱 移动端适配 | 卖家 99% 在电脑上写内容 | P2 |

## 附：MVP 验证成功标准

| 指标 | 目标 |
|------|------|
| 生成一篇内容的时间 | < 60 秒 |
| 生成内容可直接发布率（无需大改） | > 70% |
| 内部使用：likecamping.com 周产文章数 | 从 2 篇提升到 10+ 篇 |
| 外部验证：给 5 个卖家朋友试用，愿意继续用的 | ≥ 3 人 |

达到以上标准 → 启动 Phase 2（加登录 + 付费）。
